# セッションフィードバック

やり取りから得た知見を抽出し、CLAUDE.md・Skills・運用ルールへの反映が必要か判定する。

## 実行タイミング

- 作業セッションの終わり
- 新しいワークフローやツールを試した後
- 問題解決や試行錯誤を経た後

## 振り返り観点

以下の4つの観点で、やり取りを振り返る。

### 1. CLAUDE.mdへの追加

- プロジェクト全体に影響する方針・ルールの変更があったか
- 今後のやり取りで常に意識すべき内容があるか

### 2. 新規Skills作成

- 繰り返し使いそうなワークフローがあったか
- 定型化できる作業パターンがあったか

### 3. 既存Skillsの改善

- 使用したスキルに改善点があったか
- 不足している機能や観点があったか

### 4. 運用ルールの見直し

- Issue管理、Git運用、ファイル構成などで改善点があったか
- 新しい運用パターンが生まれたか

## 出力フォーマット

```markdown
## セッションフィードバック

### 1. CLAUDE.mdへの追加
- （内容があれば記載、なければ「特になし」）

### 2. 新規Skills作成
- （内容があれば記載、なければ「特になし」）

### 3. 既存Skillsの改善
- （内容があれば記載、なければ「特になし」）

### 4. 運用ルールの見直し
- （内容があれば記載、なければ「特になし」）

### 総評
（一言コメント）
```

## 注意事項

- 無理に内容を捻り出さない。本当に必要なものだけ提案する
- 「特になし」は正常な結果。毎回何かあるわけではない
- 提案する場合は具体的なアクション（何をどこに追加するか）を明記
- ユーザーの判断を仰ぐ形で提案する（勝手に変更しない）
