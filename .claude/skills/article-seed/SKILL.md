# 記事ネタストック

会話の中から記事ネタになりそうなトピックを抽出し、`ideas/article-seeds/` に保存する。

**手動発動専用**: 会話の終わりや、ネタを見つけた時に `/article-seed` で発動。

## 使い方

```
/article-seed
```

引数なしで発動。直前の会話を振り返り、記事ネタを抽出する。

## 実行手順

### 1. 会話の振り返り

直前の会話（現在のセッション）を振り返り、以下の観点で記事ネタを探す：

**ネタになりやすいパターン**:
- ユーザーの独自の知見・工夫・ノウハウ
- 「これ面白いですね」「刺さりそう」と感じた話題
- 一般的な議論に対する独自の視点・反論
- 課題→解決のストーリーがあるもの
- 複数の技術・概念を組み合わせたアイデア
- キャリア・マネジメント・働き方に関する気づき（壁打ちから生まれることが多い）

### 2. ネタの構造化

各ネタについて以下を整理：

- **仮タイトル**: キャッチーで内容が伝わるもの
- **一言**: エレベーターピッチ（1〜2文）
- **きっかけ**: どの会話/URLから発想したか
- **ポイント**: 主張・論点（箇条書き）
- **ターゲット読者**: 誰に刺さるか
- **切り口案**: 書き方のバリエーション
- **派生ネタ**: 枝葉の広がり、関連トピック

### 3. 出力フォーマット

```markdown
# 記事ネタストック: YYYY-MM-DD HH:MM

**セッション概要**: [この会話で話したことの簡潔なまとめ]

---

## ネタ1: [仮タイトル]

**一言**: [エレベーターピッチ]

### きっかけ
[どの会話/URLから発想したか]

### ポイント
- [主張1]
- [主張2]
- [主張3]

### ターゲット読者
[誰に刺さるか、どんな課題を持つ人か]

### 切り口案
1. [書き方A]: [簡単な説明]
2. [書き方B]: [簡単な説明]

### 派生ネタ
- [関連トピック1]: [一言説明]
- [関連トピック2]: [一言説明]

### ステータス
- [x] 種
- [ ] 下書き中
- [ ] 公開済み

---

## ネタ2: [仮タイトル]

（同じフォーマットで続く）

---

## 今回のセッションで見送ったネタ

（ネタになりそうだが今回は見送った話題があれば記録）

- [話題]: [見送った理由]
```

### 4. 保存

**ファイル名**: `YYYYMMDD-HHMMSS-article-seeds.md`
**保存先**: `ideas/article-seeds/`

## ネタ抽出のコツ

### 心構え: 知識を売るな、経験を売れ

知識だけなら検索やAIで手に入る。記事の価値は「なぜ自分がこれを書くのか」「どういう経験からたどり着いたか」にある。

だから **「きっかけ」が重要**：
- **経験の証明**: 自分がその話題に触れた文脈
- **読者との接点**: 同じ状況にいる人が共感できる
- **書く資格**: なぜ自分がこれを語れるのか

「誰でも書ける記事」ではなく「自分にしか書けない記事」を目指す。

### 良いネタの特徴
- **独自性**: 「みんな知ってる」ではなく「自分の経験・視点」がある
- **具体性**: 抽象論ではなく具体的なHow/Why
- **対立構造**: 「〇〇 vs △△」「〇〇は本当に必要か」
- **ギャップ**: 常識と異なる結論、意外な発見

### 枝葉の広げ方
- **深掘り**: なぜそうなのか？根本原因は？
- **横展開**: 他の分野/技術に適用できるか？
- **逆張り**: 反対意見を書いたらどうなる？
- **初心者向け**: 入門記事として書けるか？
- **上級者向け**: マニアックな詳細を書けるか？

## 注意事項

- ネタがない場合は「今回の会話では記事ネタは見つかりませんでした」と報告
- 無理にネタを作らない（質より量を追わない）
- ユーザーの発言をそのまま引用しない（プライバシー配慮）
- 重複は気にしない（ウィークリー集計で整理する前提）

## 使用例

### 会話の終わりに発動

```
（会話が一段落したところで）
/article-seed
```

### 特定の話題を深掘りしたい時

```
（面白い話題が出た直後に）
/article-seed
```
